# Code of Conduct

## Our Pledge

We are committed to making RustFinance Terminal a respectful, safe, and useful open-source community for Rust developers, quant researchers, fintech engineers, security reviewers, and curious users at every experience level.

## Expected Behavior

- Be kind, patient, and technically honest.
- Assume good intent, but accept correction quickly.
- Critique code and ideas, not people.
- Keep trading and security discussions responsible.
- Disclose conflicts of interest when recommending brokers, exchanges, vendors, or paid services.
- Respect that design choices carry trade-offs; explain trade-offs clearly.

## Unacceptable Behavior

- Harassment, insults, threats, or sustained personal attacks
- Doxing or sharing private information
- Sexualized language or imagery
- Encouraging unsafe live trading, market manipulation, or regulatory evasion
- Publishing secrets, credentials, private keys, tokens, or exploit details in public issues
- Spam, deceptive promotion, or unrelated funding links

## Financial and Security Responsibility

This project is open-source research infrastructure. Community members must not present project output as financial advice, guaranteed trading performance, or production readiness without evidence. Security issues must be reported privately through the process in [SECURITY.md](SECURITY.md).

## Enforcement

Maintainers may remove comments, close issues, reject contributions, block users, or report abuse to GitHub when behavior violates this code.

To report a conduct concern, email `security@ashutosh0x.dev` with the subject `Code of Conduct violation`. Include links, screenshots, timestamps, and any other relevant context.

This code is aligned with the spirit of the Rust project community standards and the Contributor Covenant style of open-source governance.
